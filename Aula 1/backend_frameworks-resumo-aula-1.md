## 2. O que é um Framework?

Um **framework** é uma estrutura de software reutilizável que fornece uma base pronta para o desenvolvimento de aplicações, oferecendo:
- Arquitetura definida
- Conjunto de bibliotecas
- Convenções e padrões
- Ferramentas integradas

Diferente de código desenvolvido do zero, o framework **define o fluxo da aplicação**, aplicando o conceito de *Inversion of Control (IoC)*.

> No framework, não é o desenvolvedor que controla o fluxo da aplicação.  
> É o framework que chama o código do desenvolvedor.

### Exemplos de Frameworks Back-end
- Spring Boot (Java)
- NestJS (Node.js)
- Django (Python)
- Laravel (PHP)
- ASP.NET (C#)

## 3. Biblioteca x Framework

### Biblioteca
Uma biblioteca é um conjunto de funções ou classes que o desenvolvedor utiliza conforme a necessidade.

**Características:**
- O desenvolvedor chama a biblioteca
- Resolve problemas pontuais
- Não impõe arquitetura

**Exemplos:**
- Axios
- Lodash
- Jackson
- Moment.js

> Biblioteca é como uma **caixa de ferramentas**: você escolhe quando usar.

---

### Framework
Um framework define a estrutura da aplicação e controla o fluxo de execução.

**Características:**
- O framework chama o código do desenvolvedor
- Impõe padrões e arquitetura
- Centraliza decisões técnicas

> Framework é como a **planta de um prédio**: você constrói seguindo regras.

---

### Comparação Biblioteca x Framework

| Critério | Biblioteca | Framework |
|--------|-----------|----------|
| Controle do fluxo | Código do desenvolvedor | Framework |
| Arquitetura | Livre | Definida |
| Aprendizado | Mais rápido | Curva maior |
| Escalabilidade | Limitada | Alta |

## 4. Vantagens do Uso de Frameworks

- Maior produtividade
- Padronização do código
- Facilidade de manutenção
- Comunidade ativa
- Alinhamento com o mercado de trabalho

---

## 5. Desvantagens do Uso de Frameworks

- Curva de aprendizado
- Overhead de recursos
- Dependência tecnológica
- Menor controle em algumas decisões

> Framework não é uma solução universal, mas uma escolha estratégica.

---

## 7. Conclusão

Frameworks são fundamentais no desenvolvimento back-end moderno. Saber **quando e como utilizá-los** é uma competência essencial para o profissional de tecnologia.

> Framework não é escolha de gosto.  
> É escolha de problema, contexto e escala.
